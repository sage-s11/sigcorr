## Overview

Silent Location Tracking is the most common SS7 attack. An attacker sends a Send Routing Information (SRI) request to the HLR to obtain the subscriber's serving MSC/VLR, then follows up with a Provide Subscriber Information (PSI) request to get precise cell-level location data. The subscriber is never notified.

**Step 1 вЂ” SRI (SendRoutingInfo):** The attacker sends an SRI request containing the target's MSISDN (phone number). The HLR responds with the subscriber's IMSI and the address of their current serving MSC/VLR. This is a legitimate operation used in call routing, which is why it's hard to block outright.

**Step 2 вЂ” PSI (ProvideSubscriberInfo):** Using the IMSI obtained from step 1, the attacker sends a PSI request to the VLR. The VLR responds with the subscriber's current Cell ID and Location Area Code (LAC), which can be mapped to a physical location with accuracy ranging from a few hundred meters in urban areas to several kilometers in rural areas.

## What SigCorr Detects

SigCorr triggers ATK-001 when it observes an SRI followed by a PSI for the same subscriber within the configured temporal window (default: 300 seconds). The correlation engine links the two events through IMSI в†” MSISDN identity resolution, even if the SRI uses MSISDN and the PSI uses IMSI.

**Detection criteria:**
- SRI request observed with a target MSISDN
- PSI request observed with the corresponding IMSI
- Both events originate from the same source GT (Global Title) or the same network
- Events occur within the temporal correlation window

**Confidence scoring:**
- Base: 80% (two-step pattern match)
- +10% if same source GT for both requests
- +5% if temporal gap is under 30 seconds (automated tool behavior)
- -10% if source GT is in the trusted whitelist

## Real-World Impact

This attack has been demonstrated against live networks by researchers at SRLabs, Positive Technologies, and others. It requires only SS7 network access, which can be obtained through rogue MVNOs, compromised roaming hubs, or insider access. The attack is completely silent вЂ” the target receives no notification.

Notable incidents include the 2014 demonstration by Tobias Engel at 31C3 and the 2017 exploitation reported against US subscribers via networks in the Caribbean.

## Mitigation

- Deploy SS7 firewalls that filter SRI requests from unauthorized sources
- Implement GSMA FS.11 Category 1 monitoring
- Use home-routing for SRI responses to prevent IMSI disclosure
- Monitor for anomalous SRI+PSI patterns from the same originating GT
