# ATK-003: Multi-Protocol Reconnaissance

| Field | Value |
|-------|-------|
| **ID** | ATK-003 |
| **Severity** | CRITICAL |
| **Protocol** | SS7/MAP + Diameter S6a + GTPv2-C |
| **MITRE ATT&CK** | [T1422 — System Network Configuration Discovery](https://attack.mitre.org/techniques/T1422/) |
| **MITRE FiGHT** | FGT5012 — Subscriber profile manipulation |
| **GSMA** | FS.11 Cat 1 + FS.19 Cat A |

## Overview

Multi-Protocol Reconnaissance is the signature detection that makes SigCorr unique. This attack involves coordinated probing across SS7/MAP, Diameter S6a, and GTPv2-C to build a complete profile of a target subscriber. No single-protocol monitor can detect this — the individual events on each protocol look benign in isolation.

## Attack Chain

```
Attacker         HLR/HSS        MME          PGW
   │               │             │             │
   │── MAP SRI ──→│             │             │
   │←── IMSI ─────│             │             │
   │               │             │             │
   │── DIA AIR ───────────────→│             │
   │←── Auth vectors ──────────│             │
   │               │             │             │
   │── GTP CreateSession ─────────────────→│
   │←── Bearer info ───────────────────────│
```

**Step 1 — MAP SRI:** Obtain the target's IMSI and serving network location via SS7.

**Step 2 — Diameter AIR (Authentication-Information-Request):** Query the HSS via Diameter S6a to obtain authentication vectors for the subscriber. This crosses from the legacy SS7 domain into the 4G/LTE domain.

**Step 3 — GTP CreateSessionRequest:** Attempt to establish a GTP tunnel for the subscriber, potentially hijacking their data session or establishing a parallel one.

## What SigCorr Detects

SigCorr is the first open-source tool to correlate events across all three protocol families for the same subscriber. ATK-003 triggers when MAP, Diameter, and GTP events targeting the same IMSI/MSISDN are observed within the temporal window.

**Detection criteria:**
- Events from at least 2 different protocol families (SS7, Diameter, GTP)
- All events resolve to the same subscriber identity
- Events occur within the temporal correlation window
- The event sequence follows a reconnaissance → exploitation pattern

**Confidence scoring:**
- Base: 85% (multi-protocol correlation)
- +10% if all three protocol families are involved
- +5% if events are tightly sequenced (< 60 seconds apart)
- Cross-protocol flag is always set to `true`

## Why This Matters

Traditional SS7 firewalls only see MAP traffic. Diameter security monitors only see S6a/S6d. GTP monitors only see tunnel events. An attacker who spreads their reconnaissance across all three protocols stays below the detection threshold of every single-protocol tool. SigCorr's unified identity correlation is specifically designed to catch this.

## Mitigation

- Deploy cross-protocol correlation monitoring (like SigCorr)
- Implement GSMA FS.11 and FS.19 jointly, not in isolation
- Restrict roaming access at interconnect borders across all protocol layers
- Use signaling firewalls that share context between SS7, Diameter, and GTP inspection
